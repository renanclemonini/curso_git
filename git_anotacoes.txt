GIT - Banco de versões
git clone vai clonar o repositório remoto 
git commit grava alterações no repositório
git pull para atualizar o repositório local com o que esta no remoto (busca e mescla)
git push empurra as alterações do local para o remoto 

GITHUB - servidor onde hospeda o código

Configurando GIT
git config --global user.name "Renan Clemonini"
git config --global user.email "rclemon93@gmail.com"

git config init.defaultBranch -> mostra a branch padrão configurada

git config --global init.defaultBranch main -> altera a padrão para main

git config --global credential.helper cache (armazena temporariamente) // store (armazena permanente)
token tem data de expiração

git clone <link> nome personalizado -> para clonar e dar um outro nome para a pasta

git add remote origin <link>

git clone <url> --branch feature-1 --single-branch 

.gitkeep é uma convenção para marcar pastas vazias

git restore <nome-arquivo> -> restaura o arquivo no repositório local conforme a ultima atualização no repositório remoto (ele descarta todas alterações realizadas)

git commit --amend -m "mensagem para ser alterada do ultimo commit" -> altera a mensagem do ultimo commit

se fizer "git commit --amend" puro ele abre o editor de texto (nano/vim) para editar o ultimo commit... e para sair precisa clicar esc > :wq e enter

git reset --soft <hash_commit> -> restaura o commit com as mudanças antes do commit e deixa já trackeado

git reset --mixed <hash_commit> -> restaura o commit com as mudanças do commit e deixa 'destrackeado'

git reset --hard <hash_commit> restaura o commit e desconsidera as alterações excluindo elas e o commit será apagado da arvore de log

git reflog -> exibe o histórico de commits em lista

git reset <nome_arquivo> OU git restore --staged <nome_arquivo> => remove o arquivo da área de transferência pré commit 

git checkout -b teste -> trocar da branch atual, criar uma nova chamada teste e alternar pra ela 

git checkout main -> retorna pra branch main 

git branch -v -> exibe o ultimo commit de cada branch

git branch -> lista as branchs

git merge teste -> faz o merge das branchs

git branch -d teste -> vai excluir a branch teste

---- conflito de merge

fazer git pull, editar o arquivo resolvendo o conflito, fazer novo commit e enviar push

o git pull é a execução do git fetch + git merge em conjunto

git fetch origin main -> para baixar as alterações do repositório remoto mas sem fazer o merge

git diff main origin/main -> para ver as alterações

git merge origin/main -> depois de baixar as atualizações, ver a diferença, então usa para fazer o merge do local com o remoto 

git clone <url> --branch teste --single-branch -> comando para baixar apenas a branch teste, se não passar ele só clona a branch principal

git stash -> Criar uma nova branch mas sem enviar uma modificação junto (após deletar um determinado arquivo por exemplo)

git stash list -> vai mostrar a alteração que foi arquivada

git stash pop -> para trazer as alterações e excluir da pilha de memoria

git stash apply -> trazer as alterações e manter a modificação na lista do stash
